# v1-pilot-rc1 Release Body

## TAP v1-pilot-rc1

This pilot release publishes the first full TAP repository for private, permissioned tokenized asset workflows.

### What is included

- Dual-asset flagship flow covering:
  - stablecoin cash-leg workflows
  - tokenized stock lifecycle workflows
- Policy engine with:
  - versioned policies
  - deterministic policy hashes
  - settlement-time policy linkage checks
- Issuer governance controls with:
  - maker-checker approvals
  - issuer request lifecycle tracking
- Proof and attestation lanes with:
  - real `o1js` runtime verification paths
  - zkTLS-backed source integration
  - reference adapter and generic REST integration patterns
- Customer integration kit with:
  - onboarding packet
  - mapping templates
  - customer-owned dual-asset sandbox example
- Operator-facing artifacts with:
  - one-command pack runners
  - verified public transcripts
  - runbooks, pilot materials, and architecture docs

### Why this matters

TAP is built for the next phase of asset tokenization: private, permissioned, consortium-run infrastructure for both stablecoins and tokenized risk assets.

Instead of issuing assets directly into a fully public, increasingly commoditized model, TAP provides a self-hostable control plane for:

- policy-linked issuance and settlement
- verifiable off-chain source inputs
- customer- and institution-controlled integrations
- sovereign-rollup deployment patterns
- optional bridge rails to Ethereum

### Why now

The first generation of tokenization proved demand, but the fully public operating model leaves too much value on the table:

- private data becomes visible by default
- issuer workflows become harder to differentiate
- compliance remains too edge-based
- institutional network effects leak into shared public rails

The institutions that move first on private, permissioned tokenization infrastructure will have a much stronger moat around stablecoins and tokenized assets.
