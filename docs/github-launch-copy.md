## Release title

`v1-pilot-rc1`

## Release summary

This pilot release publishes the first complete TAP repository:

- dual-asset flagship flow for stablecoins and tokenized stocks
- policy versioning and settlement-time policy linkage
- maker-checker issuer workflows
- real o1js proof runtime paths
- zkTLS-backed source integration
- partner adapter framework and customer sandbox templates
- public transcript pack, runbooks, onboarding kits, and launch documentation

The repository is designed for banks, issuers, custodians, brokers, and consortium operators who want to evaluate private, permissioned tokenization infrastructure that can be self-hosted and adapted to their own sandbox or production environments.

## Release notes body

```markdown
## TAP v1-pilot-rc1

This pilot release publishes the first full TAP repository for private, permissioned tokenized asset workflows.

### What is included

- Dual-asset flagship flow covering:
  - stablecoin cash-leg workflows
  - tokenized stock lifecycle workflows
- Policy engine with:
  - versioned policies
  - deterministic policy hashes
  - settlement-time policy linkage checks
- Issuer governance controls with:
  - maker-checker approvals
  - issuer request lifecycle tracking
- Proof and attestation lanes with:
  - real `o1js` runtime verification paths
  - zkTLS-backed source integration
  - reference adapter and generic REST integration patterns
- Customer integration kit with:
  - onboarding packet
  - mapping templates
  - customer-owned dual-asset sandbox example
- Operator-facing artifacts with:
  - one-command pack runners
  - verified public transcripts
  - runbooks, pilot materials, and architecture docs

### Why this matters

TAP is designed for the next phase of asset tokenization: private, permissioned, consortium-run infrastructure that supports both stablecoin issuance and tokenized risk assets while keeping sensitive source data off-chain.

Instead of issuing assets directly into a public-chain-first operating model, TAP provides a self-hostable control plane for:

- policy-linked issuance and settlement
- verifiable off-chain inputs
- customer- and institution-controlled integrations
- sovereign-rollup deployment patterns
- optional public-chain bridge rails

### Best entry points

- Start with the main README
- Read the flagship runbook
- Review the public transcript pack
- Use the customer sandbox mapping kit to adapt TAP to your own systems
```
